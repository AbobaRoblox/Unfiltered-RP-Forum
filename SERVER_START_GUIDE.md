# Как запустить сервер (решение проблем)

## Ошибка: "EADDRINUSE: address already in use :::3000"

Эта ошибка означает, что порт 3000 уже занят. Это может быть:
1. **Сервер уже запущен** (хорошо! Сервер работает)
2. **Другой процесс использует порт 3000**

## Решение 1: Проверить, запущен ли уже сервер

Выполните в терминале:

```bash
ps aux | grep "server-mysql.js"
```

или

```bash
pgrep -f "server-mysql.js"
```

Если вы видите процесс - сервер уже запущен! ✅

## Решение 2: Найти процесс, использующий порт 3000

```bash
lsof -i :3000
```

или

```bash
netstat -tulpn | grep :3000
```

## Решение 3: Остановить старый процесс

Если нашли процесс, который использует порт 3000, остановите его:

```bash
kill <PID>
```

где `<PID>` - это номер процесса из предыдущей команды.

Или остановите все процессы Node.js:

```bash
pkill -f "node server-mysql.js"
```

## Решение 4: Использовать другой порт

Если хотите запустить второй экземпляр на другом порту:

```bash
PORT=3001 node server-mysql.js
```

Но обычно нужно использовать только один экземпляр.

## Решение 5: Если сервер уже работает (рекомендуется)

Если сервер уже запущен, вы можете просто использовать его! 

Проверьте, работает ли он:
```bash
curl http://localhost:3000
```

Или откройте в браузере: `http://localhost:3000`

## Автоматический запуск через CRON

Если у вас настроен CRON для автоматического запуска (см. CRON_SETUP.md), сервер может запускаться автоматически. В этом случае:
- Сервер уже работает
- Не нужно запускать его вручную
- Используйте существующий процесс
